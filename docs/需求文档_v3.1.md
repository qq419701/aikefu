## 一、系统概述

### 1.1 系统定位

爱客服AI智能客服系统是一款面向多行业的自动化客服解决方案，主要特点：

- **多行业支持**：支持游戏租号、电商、教育、酒店等多个行业，每个行业独立配置
- **多店铺管理**：同一行业下可设置多个店铺，共享行业知识库，各店铺独立凭证
- **二层处理架构**：知识库 → 豆包AI，按成本从低到高递进（规则引擎已于v2.1删除）
- **北京时间驱动**：所有时间戳和定时任务均使用北京时间（UTC+8）

### 1.2 适用行业

| 行业代码 | 行业名称 | 主要场景 |
|---------|---------|---------|
| game_rental | 游戏租号 | 拼多多/淘宝游戏账号租赁 |
| ecommerce | 电商客服 | 通用电商商品销售 |
| education | 教育培训 | 课程咨询、报名、退款 |
| hotel | 酒店民宿 | 预订、退款、投诉 |
| 自定义 | 用户自定义 | 任意行业 |

### 1.3 核心指标目标

| 指标 | 目标值 | 说明 |
|------|-------|------|
| AI自动解决率 | ≥ 90% | 上线1个月后 |
| 退款驳回成功率 | ≥ 70% | 游戏租号行业 |
| 月运行成本 | ≤ ¥20 | 豆包API费用 |
| 节省人工成本 | ≥ ¥8000/月 | 对比1名全职客服 |

## 二、功能需求

### 2.1 多行业管理

**功能描述**：
- 支持添加任意行业，每个行业有独立的：
  - AI系统提示词（角色定义）
  - 知识库问答对
  - 关键词规则
  - 功能开关（自动回复、图片识别、情绪识别）
- 行业之间数据完全隔离
- 管理员可管理所有行业，操作员只能管理绑定的行业

**多行业设计说明**：
> 示例：游戏租号行业可以绑定多个店铺（店铺A、B、C），这些店铺共同使用「游戏租号」行业的知识库。
> 其他行业（如电商）的知识库和规则不会出现在游戏租号的配置界面。

### 2.2 多店铺管理

**功能描述**：
- 一个行业下可创建多个店铺
- 店铺配置包括：
  - 所属行业（确定使用哪个知识库）
  - 电商平台类型（拼多多、淘宝、京东）
  - 平台API凭证（client_id, client_secret）
  - 是否启用自动回复
  - 是否启用自动换号（游戏租号专用）
  - u号租平台账号密码（游戏租号专用）
  - 自定义AI提示词（覆盖行业默认）

### 2.3 二层AI处理架构（v2.1起）

#### 意图规则（第零层，0成本0延迟）

- **目标覆盖率**：30%+
- **处理方式**：关键词匹配
- **匹配结果**：返回意图码 + 插件动作码 + 立即回复话术
- **纯文字回复**：`action_code` 为空时，`auto_reply_tpl` 直接回复买家（0成本0延迟）
- **插件分流**：有 `action_code` 时创建 PluginTask 下发给客户端执行

#### 第一层：知识库（0成本）

- **目标覆盖率**：60%
- **处理方式**：关键词 + 相似度检索
- **相似度算法**：关键词权重60% + 词汇重叠率40%
- **阈值**：默认0.6（可配置）
- **MaxKB**：可选接入语义检索，命中率约85%

#### 第二层：豆包AI（有成本）

- **目标覆盖率**：35%（兜底）
- **处理方式**：调用火山方舟API
- **缓存机制**：相同问题24小时内复用缓存，节省约80%费用
- **模型选择**：doubao-lite-32k（文字）+ doubao-pro-32k（退款/情绪）+ doubao-vision-pro-32k（图片）
- **估算费用**：约¥0.001/次，月均¥20以内

### 2.4 自动换号（游戏租号专用）

**触发条件**：
- 买家发送包含"换号"、"换账号"等关键词
- 意图规则匹配后触发 `auto_exchange` 动作

**处理流程**：
1. 意图规则识别换号请求
2. 回复买家确认信息（auto_reply_tpl）
3. 下发 PluginTask 给客户端执行换号
4. 客户端完成后上报，服务端通过 done_reply_tpl 通知买家新账号信息

### 2.5 退款自动处理

**自动驳回流程**：
1. 意图规则识别退款关键词（`refund` intent）
2. 若无插件接管，豆包-pro 做退款决策（approve/reject/human）
3. 发送退款引导话术
4. 记录退款处理结果（用于统计驳回率）

### 2.6 图片识别（豆包Vision）

**功能**：
- 识别买家发送的截图（如登录失败截图）
- 自动分析问题类型
- 生成处理建议

**启用方式**：行业配置 → 开启「图片识别」

### 2.7 情绪识别

**情绪级别**：

| 级别 | 描述 | 处理策略 |
|-----|------|---------|
| 0 | 正常 | 正常回复 |
| 1 | 轻微不满 | 语气温柔，耐心回复 |
| 2 | 中度不满 | 深度道歉，提出解决方案 |
| 3 | 严重不满 | 立即转人工，发送安抚话术 |
| 4 | 危机（投诉威胁） | 紧急转人工，记录黑名单观察 |

### 2.8 恶意买家黑名单

- **共享范围**：同一行业下所有店铺共享黑名单
- **级别**：观察（1）、警告（2）、封禁（3）
- **自动检测**：定时任务检查退款次数超阈值（默认3次/30天）
- **手动添加**：管理员可手动加入
- **处理方式**：黑名单买家的消息自动转人工处理

### 2.9 实时消息监控

- 实时显示所有买家消息
- 按店铺/状态/日期筛选
- 待人工处理消息醒目提示
- 人工处理完成一键标记

### 2.10 数据统计报表

**每日自动统计**（北京时间凌晨1点）：
- 消息总量
- 二层处理分布（知识库/AI）+ 意图回复/插件命中
- AI自动解决率
- Token消耗和费用
- 退款/换号次数

## 四、API接口

### 4.1 消息Webhook

**接口**：`POST /api/webhook/message`

**说明**：接收平台推送的买家消息，触发三层处理

**请求格式**：
```json
{
    "shop_id": 1,
    "buyer_id": "买家平台ID",
    "buyer_name": "买家昵称",
    "order_id": "订单号",
    "content": "消息内容",
    "msg_type": "text",
    "image_url": ""
}
```

**响应格式**：
```json
{
    "success": true,
    "reply": "AI回复内容",
    "process_by": "intent_reply|plugin|knowledge|ai|human",
    "needs_human": false,
    "emotion_level": 0,
    "action": ""
}
```

### 4.2 健康检查

**接口**：`GET /api/health`

**响应**：
```json
{
    "status": "ok",
    "system": "爱客服AI智能客服系统",
    "version": "v2.1.0",
    "time": "2026-03-11 20:00:00",
    "timezone": "Asia/Shanghai (北京时间)"
}
```

## v3.2 新增需求（2026-03-11）

### 3.2.1 规则引擎删除
- 删除规则引擎（`rules` 表、相关路由、模板、模块）
- 已有规则引擎规则迁移说明：动作类（换号/退款）→ 意图规则，FAQ类 → 知识库

### 3.2.2 意图规则纯文字回复
- 意图规则新增纯文字回复能力
- `action_code` 为空时，`auto_reply_tpl` 直接回复买家
- 支持变量替换：`{buyer_name}`、`{order_id}`
- 统计标识：`process_by='intent_reply'`

### 3.2.3 系统设置页面
- 新增 `/settings/` 系统设置页面
- 新增 `SystemConfig` 数据库模型（KV存储）
- 支持4个分组：AI参数、知识库设置、系统行为、安全设置
- 支持豆包API和MaxKB连接测试

### 3.2.4 消息管理优化
- 消息表新增 `direction='out'` 存储AI/系统回复
- 消息管理页支持气泡对话视图（按买家分组）
- 支持一键将AI回复加入学习中心

### 3.2.5 控制台健康状态
- 控制台新增健康状态区块（30秒自动刷新）
- 包含：插件在线状态、MaxKB连接、豆包API状态、命中率饼图

### 3.2.6 页面帮助系统
